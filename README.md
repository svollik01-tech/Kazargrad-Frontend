# Казарьград — сайт гостевого комплекса

Одностраничный лендинг гостевого комплекса «Казарьград» + встроенная админ‑панель
(CMS‑подобный редактор) для управления всем содержимым сайта без правки кода.

Стек: **React 19 + TypeScript + Vite + Tailwind CSS + React Router + Zustand + Framer Motion**.

## Запуск

```bash
npm install
npm run dev        # дев-сервер (http://localhost:5173)
npm run build      # сборка в dist/ (tsc -b && vite build)
npm run preview    # предпросмотр собранной версии
npm run lint       # ESLint
npm run typecheck  # проверка типов
```

## Структура

```
src/
  pages/
    HomePage.tsx          публичный лендинг (собирает секции)
    admin/                страницы админ-панели
      LoginPage.tsx       вход (мок-авторизация)
      Dashboard.tsx       обзор + экспорт/импорт/сброс контента
      *Editor.tsx         редакторы каждой секции
  components/
    public/               секции публичного сайта
    admin/                каркас и переиспользуемые контролы админки
    ui/                   общие UI-компоненты (Button, Toast, …)
  store/contentStore.ts   реактивное хранилище контента (Zustand)
  services/               слой работы с бэкендом (контент, авторизация, картинки)
  data/defaultContent.ts  заводской контент (сид + сброс)
  types/content.ts        единая модель контента сайта
server/                   бэкенд (Node + Express): контент + загрузка картинок
deploy/nginx.conf         конфиг nginx (раздача dist/, /uploads/, прокси /api/)
docker-compose.yml        запуск бэкенда и nginx
```

## Админ-панель

Доступ: **`/admin`** (защищённый маршрут; неавторизованных перебрасывает на `/admin/login`).

**Демо-доступ:** логин `admin`, пароль `admin123`.

Публичный сайт и админка читают/пишут одно и то же хранилище, поэтому изменения
в админке сразу видны на сайте. Редактируются все секции: главный экран, преимущества,
дома, территория, детям, развлечения, удобства, услуги/цены, праздники, бронирование,
отзывы, FAQ, контакты, подвал, фотогалерея, а также SEO и видимость разделов.

На дашборде есть экспорт/импорт всего контента в JSON и сброс к заводским значениям.

## Бэкенд и хранилище

Контент и картинки хранятся **на сервере** и общие для всех посетителей (с любого
устройства). Весь обмен изолирован в трёх сервисах фронтенда:

- **Контент** — `src/services/contentService.ts` → `GET/PUT /api/content`.
- **Авторизация** — `src/services/authService.ts` → `POST /api/login` (подписанный токен).
- **Изображения** — `src/services/imageService.ts` → `POST /api/images` (возвращает `/uploads/<id>`).

Бэкенд — `server/index.js` (Node + Express + Multer): хранит контент в
`data/content.json`, картинки в `uploads/`, выдаёт и проверяет токен авторизации.
В localStorage браузера теперь лежит только токен сессии.

### Запуск в продакшене (Docker)

```bash
npm run build                       # собрать фронтенд в dist/
docker compose up -d --build        # поднять бэкенд (api) + nginx
```

`docker-compose.yml` поднимает два сервиса:

- **api** — бэкенд; данные и картинки лежат в volume `./data` и `./uploads`
  (переживают пересоздание контейнеров).
- **nginx** — раздаёт `dist/`, картинки из `/uploads/` и проксирует `/api/` на бэкенд.

Сайт открывается на `http://<сервер>:8002`.

> ⚠️ **Перед продакшеном** в `docker-compose.yml` смените `AUTH_SECRET` на длинную
> случайную строку и поменяйте `ADMIN_LOGIN` / `ADMIN_PASSWORD`. Секрет подписывает
> токены — кто его знает, тот может войти в админку.

### Локальная разработка

```bash
# терминал 1 — бэкенд (создаст server/data и server/uploads)
cd server && npm install && npm start

# терминал 2 — фронтенд (vite проксирует /api и /uploads на :3001)
npm run dev
```
